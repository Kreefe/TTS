## 🎮 Новые возможности

### 1. Прогресс-бары

Теперь вы видите детальный прогресс генерации:

```
🎤 Генерация аудио:
Обработка: 45%|████████████▌             | 2304/5120 [02:15<02:45, 17.0строк/s]
```

**Информация в прогресс-баре:**

- Процент выполнения
- Количество обработанных/всего строк
- Прошедшее время
- Оставшееся время (оценка)
- Скорость обработки (строк/сек)

### 2. Многопоточная генерация

Аудио генерируется параллельно в нескольких потоках, что значительно ускоряет процесс.

**По умолчанию:** 4 потока

**Настройка количества потоков:**

```bash
# Использовать 8 потоков (для мощных систем)
python ttsp.py --input ranobe.txt --workers 8

# Использовать 2 потока (для слабых систем)
python ttsp.py --input ranobe.txt --workers 2
```

### 3. Улучшенная сборка чанков

Теперь сборка финального MP3 также отображается с прогресс-баром:

```
📦 Собираю финальный файл из 10 кусков...
Сборка: 100%|████████████████████████████| 10/10 [00:05<00:00, 1.85чанк/s]
```

## ⚙️ Настройка производительности

### Рекомендации по количеству потоков

| Система | CPU/GPU      | Рекомендуемые потоки         |
| ------- | ------------ | ---------------------------- |
| Слабая  | 2-4 ядра CPU | `--workers 2`                |
| Средняя | 4-8 ядер CPU | `--workers 4` (по умолчанию) |
| Мощная  | 8+ ядер CPU  | `--workers 6-8`              |
| С GPU   | NVIDIA GPU   | `--workers 4-6`              |

### Настройка размера чанка

В [`ttsp.py`](ttsp.py:37) можно изменить `CHUNK_SIZE`:

```python
CHUNK_SIZE = 512  # По умолчанию

# Для систем с большим объемом RAM:
CHUNK_SIZE = 1024  # Меньше чанков = быстрее сборка

# Для систем с малым объемом RAM:
CHUNK_SIZE = 256  # Больше чанков = меньше потребление памяти
```

## 🔧 Технические детали

### Архитектура многопоточности

```
Входной текст (5000 строк)
         ↓
Разбивка на чанки по 512 строк
         ↓
┌────────┴────────┬────────┬────────┐
│ Поток 1         │ Поток 2│ Поток 3│ Поток 4
│ Строки 0-511    │ 512-   │ 1024-  │ 1536-
│ → chunk_0000.wav│ 1023   │ 1535   │ 2047
└────────┬────────┴────────┴────────┘
         ↓
Сборка чанков в правильном порядке
         ↓
Экспорт в MP3
```

### Thread-safe операции

- Каждый поток работает с независимым диапазоном строк
- Прогресс-бар обновляется thread-safe через `tqdm`
- Чанки сохраняются с уникальными именами (`chunk_0000.wav`, `chunk_0001.wav`, ...)
- Финальная сборка происходит после завершения всех потоков

### Оптимизация памяти

Многопоточность **не увеличивает** потребление памяти, так как:

1. Каждый поток обрабатывает только свой диапазон строк
2. Аудио сразу сбрасывается на диск в WAV-файл
3. Переменные очищаются после завершения потока

## 🐛 Устранение неполадок

### Проблема: Ошибка "No module named 'tqdm'"

**Решение:**

```bash
pip install tqdm
```

### Проблема: Медленная генерация даже с многопоточностью

**Возможные причины:**

1. **Bottleneck на GPU** — модель TTS может не поддерживать параллельную генерацию на GPU
2. **Медленный диск** — сохранение чанков на HDD замедляет процесс

**Решение:**

- Используйте меньше потоков: `--workers 2`
- Переместите проект на SSD
- Увеличьте `CHUNK_SIZE` для уменьшения количества операций записи

### Проблема: Высокое потребление CPU

**Решение:**
Уменьшите количество потоков:

```bash
python ttsp.py --input ranobe.txt --workers 2
```

### Проблема: Прогресс-бар не отображается

**Причина:** Вывод перенаправлен или терминал не поддерживает ANSI-коды

**Решение:**
Используйте современный терминал (Windows Terminal, iTerm2, GNOME Terminal)
