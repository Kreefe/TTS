## 🎯 Описание проекта

QwenTTS — это автоматизированная система для создания аудиокниг из текстовых источников. Проект состоит из двух основных компонентов:

1. **Парсер** ([`parser.py`](parser.py:1)) — автоматически загружает главы с сайта Ranobelib, обрабатывает текст и подготавливает его для озвучки
2. **TTS-движок** ([`ttsp.py`](ttsp.py:1)) — преобразует текст в речь с использованием Silero TTS и опционально отправляет результат в Telegram

### Ключевые преимущества

- ✅ **Полная автоматизация**: от загрузки текста до готового аудиофайла
- ✅ **Оптимизация памяти**: обработка больших файлов без переполнения RAM
- ✅ **Высокое качество**: использование нейросетевой модели Silero TTS v5
- ✅ **Гибкость**: поддержка различных режимов работы и настроек
- ✅ **Надёжность**: механизм повторных попыток при сбоях загрузки

## 🚀 Основные возможности

### Парсер текстов ([`parser.py`](parser.py:1))

- 🌐 **Автоматическая загрузка глав** с сайта Ranobelib через Playwright
- 🔄 **Механизм повторных попыток** (до 10 попыток на главу)
- 🔢 **Конвертация цифр в слова** для корректного произношения TTS
- 📝 **Очистка текста** от лишних переносов строк
- ⚡ **Асинхронная обработка** для максимальной скорости
- 🎯 **Автоматический запуск озвучки** после парсинга

### TTS-движок ([`ttsp.py`](ttsp.py:1))

- 🎙️ **Синтез речи** с использованием Silero TTS v5 (русский язык)
- 💾 **Оптимизация памяти** через чанкование (обработка по 512 строк)
- 📚 **Два режима работы**:
  - Один большой MP3 файл
  - Разбивка на отдельные главы
- 🎵 **Высокое качество аудио**: 48 кГц, 128 kbps MP3
- 🚀 **GPU-ускорение**: автоматическое использование CUDA при наличии
- 📤 **Интеграция с Telegram** через Telethon API
- 🌐 **Поддержка SOCKS5 прокси**

## 📦 Установка

### Системные требования

- **Python**: 3.7 или выше
- **ОС**: Windows, Linux, macOS
- **RAM**: минимум 4 ГБ (рекомендуется 8 ГБ)
- **GPU** (опционально): NVIDIA с поддержкой CUDA для ускорения

### Шаг 1: Клонирование репозитория

```bash
git clone <repository-url>
cd QwenTTS
```

### Шаг 2: Установка Python-зависимостей

```bash
pip install -r requirements.txt
```

## 🔧 Технические детали

### Парсер ([`parser.py`](parser.py:1))

#### Функция [`clean_and_textify()`](parser.py:15)

Обрабатывает текст для корректной озвучки:

```python
def clean_and_textify(text):
    # 1. Цифры → слова: "123" → "сто двадцать три"
    text = re.sub(r'\d+', lambda m: num2words(int(m.group(0)), lang='ru'), text)
    
    # 2. Убираем лишние переносы строк
    text = re.sub(r'\n\s*\n+', '\n\n', text)
    
    return text.strip()
```

Загружает главу с механизмом повторных попыток:

- **Стратегия**: перезагрузка страницы при неудаче
- **Селектор текста**: `p.node-paragraph`
- **Селектор заголовка**: `h1`

## 💾 Оптимизация памяти

### Проблема

При озвучке больших текстов (тысячи строк) накопление аудио в памяти приводит к переполнению RAM.

### Решение: Чанкование

Каждые **512 строк** (настраивается через [`CHUNK_SIZE`](ttsp.py:37)):

1. Накопленное аудио сбрасывается на диск в WAV-файл
2. Переменная `combined_audio` очищается
3. RAM освобождается

```python
if (i + 1) % CHUNK_SIZE == 0 or (i + 1) == total:
    chunk_name = os.path.join(TEMP_DIR, f"chunk_{len(chunk_files)}.wav")
    combined_audio.export(chunk_name, format="wav")
    chunk_files.append(chunk_name)
    combined_audio = AudioSegment.empty()  # Очистка памяти
```

### Финальная сборка

После обработки всех строк чанки объединяются в один MP3:

```python
final_audio = AudioSegment.empty()
for chunk in chunk_files:
    final_audio += AudioSegment.from_wav(chunk)
final_audio.export(OUTPUT_MP3, format="mp3", bitrate="128k")
```

### Преимущества

- ✅ Обработка файлов любого размера
- ✅ Стабильное потребление RAM
- ✅ Автоматическая очистка временных файлов

## 🛠️ Устранение неполадок

### Проблема: "Файл не найден" при запуске парсера

**Решение:**
```bash
# Убедитесь, что Playwright установлен
playwright install chromium
```

### Проблема: Ошибка "API_ID not found"

**Решение:**
1. Проверьте наличие файла `.env` в корне проекта
2. Убедитесь, что в `.env` указаны `API_ID` и `API_HASH`
3. Перезапустите скрипт

### Проблема: Медленная генерация аудио

**Решение:**
1. Проверьте наличие GPU: `torch.cuda.is_available()`
2. Установите CUDA-версию PyTorch:
   ```bash
   pip install torch --index-url https://download.pytorch.org/whl/cu118
   ```

### Проблема: Переполнение памяти

**Решение:**
Уменьшите [`CHUNK_SIZE`](ttsp.py:37) в [`ttsp.py`](ttsp.py:1):
```python
CHUNK_SIZE = 256  # Вместо 512
```

### Проблема: Главы не загружаются (таймаут)

**Решение:**
1. Проверьте интернет-соединение
2. Увеличьте таймаут в [`parser.py`](parser.py:65):
   ```python
   await page.goto(url, wait_until="commit", timeout=5000)  # 5 секунд
   ```

### Проблема: FFmpeg не найден

**Решение:**
- **Windows**: Поместите `ffmpeg.exe` в папку проекта
- **Linux/macOS**: Установите через пакетный менеджер

## ❓ FAQ

### Какие голоса доступны?

Silero TTS v5 для русского языка поддерживает:
- `kseniya` (женский, по умолчанию)
- `aidar` (мужской)
- `baya` (женский)
- `eugene` (мужской)
- `xenia` (женский)

Измените в [`ttsp.py`](ttsp.py:31):
```python
SPEAKER = 'aidar'  # Мужской голос
```

### Можно ли озвучивать на других языках?

Да, Silero TTS поддерживает несколько языков. Измените в [`ttsp.py`](ttsp.py:29-30):
```python
LANGUAGE = 'en'      # Английский
MODEL_ID = 'v3_en'   # Модель для английского
```

### Как отправить готовый MP3 в Telegram?

Раскомментируйте код в [`ttsp.py`](ttsp.py:314-319):
```python
if ready:
    choice = input("\n❓ Отправить файл в Telegram? (y/n): ").strip().lower()
    if choice == 'y':
        await send_via_telethon(OUTPUT_MP3)
```

### Сколько времени занимает озвучка?

Зависит от:
- **Длины текста**: ~1-2 минуты на 100 строк (CPU)
- **Оборудования**: GPU ускоряет в 3-5 раз
- **Частоты дискретизации**: 48 кГц медленнее, чем 24 кГц

### Можно ли изменить качество MP3?

Да, измените битрейт в [`ttsp.py`](ttsp.py:175):
```python
final_audio.export(output_file, format="mp3", bitrate="192k")  # Выше качество
```

### Как работает механизм повторных попыток?

При загрузке главы парсер делает до 10 попыток:
1. Загружает страницу
2. Ждёт появления текста (2 секунды)
3. При неудаче — перезагружает страницу (F5)
4. После 10 неудач — останавливает программу

### Безопасно ли хранить API ключи?

Файл `.env` автоматически исключён из Git через [`.gitignore`](.gitignore:1). Никогда не публикуйте `.env` в открытых репозиториях.

### Можно ли обрабатывать несколько книг параллельно?

Да, запустите несколько экземпляров скрипта в разных терминалах с разными входными файлами:

```bash
# Терминал 1
python ttsp.py --input book1.txt --name book1

# Терминал 2
python ttsp.py --input book2.txt --name book2
```
